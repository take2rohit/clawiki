# World Models

> **One sentence** — Ha & Schmidhuber's VAE + MDN-RNN agent achieves 906 ± 21 on CarRacing-v0 (first reported solution) and 1092 ± 556 on DoomTakeCover-v0 by training a compact controller entirely inside a hallucinated dream environment generated by its own world model.

**Authors:** David Ha, Jürgen Schmidhuber | **Venue:** NeurIPS 2018 | **arXiv:** [1803.10122](https://arxiv.org/abs/1803.10122)

---

## Problem & Motivation

Model-free RL agents must learn policies through expensive direct interaction with the environment, and the RL credit assignment problem makes training large neural networks as policy networks impractical. Prior work typically used small networks (10³–10⁶ parameters) as controllers, limiting representational capacity. Separately, large generative models could capture rich world representations but were not being used to directly train agents. The key opportunity was to decouple the capacity problem: use a large unsupervised world model to absorb environment complexity, and then train only a tiny controller on top of it — shifting the credit assignment burden to a simple, low-dimensional search problem solvable by evolution.

---

## Core Idea

The authors realized that a generative world model trained unsupervised on environment observations can serve as a complete substitute for the real environment during policy learning. By separating the agent into a large Vision+Memory world model (V+M) and a tiny Controller (C), the complex representational learning is handled by V and M, leaving C so small (867 parameters) that evolution strategies can optimize it efficiently. The most striking result is that C can be trained entirely inside M's "dream" — a hallucinated rollout — and the resulting policy transfers back to the actual environment.

---

## How It Works

### Overview

At each timestep: raw pixel observation → VAE encodes to latent z → MDN-RNN maintains hidden state h predicting future z → Controller maps [z, h] → action → action fed back into environment and MDN-RNN. The world model (V + M) is trained unsupervised; the controller C is trained separately by CMA-ES.

### Vision Model (V) — Convolutional VAE

A Convolutional Variational Autoencoder (ConvVAE) compresses each 64×64×3 RGB frame into a low-dimensional latent vector z. The encoder uses 4 convolutional layers (stride 2, ReLU activations) producing a bottleneck from which μ and σ (each of size N_z) are output; z is sampled from N(μ, σI). The decoder uses 4 deconvolutional layers to reconstruct the frame. Loss is reconstruction L² + KL divergence. For CarRacing N_z = 32; for Doom N_z = 64. Parameter count: ~4.3–4.4M. The Gaussian prior on z also helps robustness when M generates novel z vectors during dreaming.

### Memory Model (M) — MDN-RNN

An LSTM combined with a Mixture Density Network (MDN) output layer models the temporal dynamics of the latent space. At each step it receives (a_t, z_t) and produces a probability distribution over the next latent vector: P(z_{t+1} | a_t, z_t, h_t) as a mixture of Gaussians. For Doom it additionally predicts done_t (agent death) as a binary event. LSTM hidden size: 256 units (CarRacing), 512 units (Doom); 5 Gaussian mixture components; diagonal covariance. A temperature parameter τ controls uncertainty during sampling. Parameter count: ~422K–1.68M.

### Controller (C)

A single linear layer mapping the concatenation [z_t, h_t] directly to action a_t:

a_t = W_c [z_t h_t] + b_c

This minimal design (only 867 parameters for CarRacing, 1,088 for Doom) is intentional: it concentrates complexity in V and M, making C's parameter space small enough for CMA-ES to optimize without backpropagation through the environment.

### Training

**Stage 1 — Data collection:** 10,000 random rollouts collected from a random policy to build a dataset of observations and actions.

**Stage 2 — Train V:** ConvVAE trained for 1 epoch on collected frames, minimizing L² reconstruction + KL loss. Trained on a single GPU in under an hour.

**Stage 3 — Train M:** MDN-RNN trained for 20 epochs on sequences of (a_t, z_t) pairs (teacher forcing with sampled z ~ N(μ, σ)).

**Stage 4 — Optimize C:** CMA-ES with population size 64, each agent evaluated 16 times with different random seeds. Fitness = average cumulative reward. Run on a 64-core machine (~1800–2500 generations for convergence).

### Inference

At test time, the agent runs V and M in real time alongside the actual environment: each frame is encoded by V to z_t, C reads [z_t, h_t] and outputs action a_t, M updates its hidden state. For dream-only training (VizDoom experiment), C is optimized entirely inside hallucinated rollouts generated by M — no VAE needed during dreaming since the agent operates entirely in latent space. The learned policy is then transferred directly to the real environment.

## Results

### CarRacing-v0

| Method | Avg. Score (100 trials) |
|--------|------------------------|
| **Full World Model (V+M+C)** | **906 ± 21** |
| V Model only (z→a, linear) | 632 ± 251 |
| V Model + hidden layer | 788 ± 141 |
| ceobillionaire (Gym leaderboard) | 838 ± 11 |
| A3C Discrete (Khan & Elibol, 2016) | 652 ± 10 |
| A3C Continuous (Jang et al., 2017) | 591 ± 45 |
| DQN (Prieur, 2017) | 343 ± 18 |

The task is considered "solved" at 900+ average over 100 consecutive trials. The full world model is the first reported solution. The gap between V-only (632) and V+M (906) proves the memory component h_t is essential — it encodes predictive information about the future that z_t alone lacks.

### DoomTakeCover-v0

| Method | Survival (time steps, 100 trials) |
|--------|-----------------------------------|
| **World Model (trained in dream, τ=1.15)** | **1092 ± 556** |
| Gym Leaderboard | 820 ± 58 |
| Random Policy | 210 ± 108 |

The agent was trained entirely inside the dream environment (DoomRNN) without ever touching the real environment during controller optimization. Transfer to the actual environment exceeded the required threshold of 750 time steps and beat the gym leaderboard.

### Temperature Ablation (DoomTakeCover)

| Temperature τ | Virtual Score | Actual Score |
|--------------|---------------|--------------|
| 0.10 | 2086 ± 140 | 193 ± 58 |
| 0.50 | 2060 ± 277 | 196 ± 50 |
| 1.00 | 1145 ± 690 | 868 ± 511 |
| 1.15 | 918 ± 546 | **1092 ± 556** |
| 1.30 | 732 ± 269 | 753 ± 139 |

This ablation reveals a critical "dream exploitation" failure mode: at low τ the world model is near-deterministic and the controller finds adversarial exploits (e.g., fireballs that never fire) that work in the dream but fail in reality. Higher τ injects uncertainty that prevents exploitation, but too high makes the dream unlearnable. τ = 1.15 is the sweet spot.

### Ablations

Removing M (using z only, no hidden state h) drops CarRacing score from 906 ± 21 to 632 ± 251 (30% reduction), proving that temporal prediction — not just spatial compression — is load-bearing for the task. Adding a hidden layer to the z-only controller raises it to 788 ± 141, still well below the full model, confirming the benefit comes from M's predictive content rather than controller capacity.

## Comparison to Prior Work

| Method | World Model | Controller Training | Dream Training |
|--------|------------|--------------------|----|
| **World Models (this paper)** | VAE + MDN-RNN | CMA-ES (evolution) | Yes (full) |
| PILCO (Deisenroth & Rasmussen, 2011) | Gaussian Process | Policy gradient | No |
| Oh et al. 2015 / Chiappa et al. 2017 | FNN/RNN frame prediction | Separate RL | No |
| Schmidhuber 1990a/b | RNN world model | RNN controller (joint) | Conceptual |
| Nagabandi et al. 2017 | Neural network dynamics | Model-free fine-tuning | No |

**PILCO:** Uses Gaussian processes for dynamics, scales poorly to high-dimensional pixel observations; limited to low-D state spaces. World Models operates directly on raw pixels via the VAE.

**Oh et al. / Chiappa et al.:** Learn to predict future frames with FNNs/RNNs but do not use the world model to replace the real environment for training — the controller still trains in the real env.

**Schmidhuber 1990a/b:** The conceptual ancestor — an RNN world model + RNN controller trained jointly. World Models separates concerns and uses modern deep learning components (ConvVAE, LSTM-MDN) plus evolution for the controller.

**Nagabandi et al. 2017:** Combines model-based initialization with model-free fine-tuning; still requires real environment interaction for policy refinement. World Models avoids this for the VizDoom task entirely.

## Strengths
- First demonstrated that a policy can be trained entirely inside a hallucinated dream and successfully transferred to the real environment
- Clean modular architecture (V/M/C separation) that matches cognitive/neuroscience intuitions about perception, memory, and action
- CMA-ES for controller optimization sidesteps the credit assignment problem entirely for the small-parameter C
- Temperature parameter τ provides an explicit knob controlling dream realism vs. exploitability
- Achieved first reported solution to CarRacing-v0 (score ≥ 900) from raw pixels
- Extremely compute-efficient controller training — 867 parameters optimizable on a single 64-core machine

## Weaknesses & Limitations
- VAE trained unsupervised may encode task-irrelevant features (e.g., wall tiles in Doom) and miss task-relevant ones (road tiles in CarRacing)
- World model exploitation / "cheating" is a fundamental failure mode: the controller will discover adversarial exploits if the world model is imperfect
- LSTM capacity is limited; catastrophic forgetting may occur as the agent explores new regions of the environment
- Sequential three-stage training (V → M → C) cannot jointly optimize the world model for task relevance
- Only tested on two relatively simple RL environments; scalability to more complex domains was not demonstrated
- The iterative training procedure needed for harder tasks was described but not experimentally validated

## Key Takeaways
- A world model (VAE + MDN-RNN, ~4.8M parameters) can serve as a complete substitute for the real environment when training a controller (867 parameters), enabling "dream training" with successful real-world transfer
- The memory component h_t is critical: removing it drops CarRacing from 906 to 632 (a 30% performance gap), proving temporal prediction is as important as spatial compression
- Dream exploitation is real: temperature τ must be tuned to prevent the controller from finding model exploits; τ = 1.15 gave the best real-world transfer for DoomTakeCover
- Evolution strategies (CMA-ES) are a practical alternative to RL for optimizing tiny controllers when world model features are available as inputs
- The V/M/C decomposition is foundational: nearly all subsequent world model papers — [[hafner-2019-icml]] ([PlaNet](../papers/hafner-2019-icml.md)), [[hafner-2021-iclr]] ([DreamerV2](../papers/hafner-2021-iclr.md)), [[hafner-2023-arxiv]] ([DreamerV3](../papers/hafner-2023-arxiv.md)) — inherit this separation of perception, dynamics, and control
